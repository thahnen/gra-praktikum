18 /*
19  * C API version, selectable by setting ROYALE_C_API_VERSION to one of the following values:
20  *
21  * 220 - C API as of Royale v2.2.0
22  * 300 - C API as of Royale v3.0.0 (includes support for mixed-mode)
23  * Mixed mode added StreamIds to many functions, including one callback (the exposure listener)
24  * 320 - C API as of Royale v3.2.0
25  * added royale_get_version_with_build_and_scm_revision() (see royaleCAPI.h)
26  * added functions to shift and retrieve the lens center
27  * 330 - C API as of Royale v3.3.0
28  * added royale_camera_device_get_number_of_streams
29  * added royale_camera_device_set_external_trigger
30  * 31000 - C API as of Royale v3.10.0
31  * added royale_camera_device_write_data_to_flash
32  *
33  * Older C API versions were already deprecated in previous releases and have now been removed.
34  *
35  * Starting with Royale v3.0.0, C API compatibility is selected by defining ROYALE_C_API_VERSION
36  * to one of the above values. To aid in migration, the latest API version is also made available
37  * using alternative names (with a "_new_api" suffix added) if the preprocessor symbol
38  * ROYALE_C_API_MIGRATION is defined.
39  *
40  * A possible migration strategy would be:
41  * - define ROYALE_C_API_VERSION=220 and ROYALE_C_API_MIGRATION (to any value)
42  * - compile the code. The compiler should warn about deprecated functions.
43  * - fix all warnings by adapting the code to the new version, using the alternate function name
44  * with "_new_api" suffix
45  * - after all warnings are fixed, unset ROYALE_C_API_MIGRATION and ROYALE_C_API_VERSION
46  * - remove the "_new_api" suffix from the C API calls in the code
47  *
48  * In previous releases, the C API version offered could be selected using the macros
49  * ROYALE_NEW_API_2_2_0 and ROYALE_FINAL_API_2_2_0. Defining any of these will now
50  * lead to a compiler error (close to a piece of documentation telling you how to fix this).
51  */
52

85  * If there are errors or warning regarding the API functions, define
